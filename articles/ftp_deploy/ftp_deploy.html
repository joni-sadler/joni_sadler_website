When I first set up this website I was making updates directly through my hosting service. Eventually I moved to an FTP app (FileZilla) to smooth out the process, and while this was a step in the right direction it still felt like there was room for improvement. A thought struck: what if I could update my site automatically every time I push content to GitHub?

The first step in implementing this was looking up FTP Deploy in GitHub Actions.

Following the provided instructions, I created a .yml file in VS Code and added some preliminary info:

The next step was to modify this original info with my username, server details, and password. I didn't want to store the password directly in the .yml file for security reasons. GitHub secrets to the rescue!

To set up a GitHub secret, navigate to your project repository, go to 'settings', and you will see 'secrets' in the menu displayed on the left side of your page:

In 'secrets' you will see two options: Environment secrets and Repository secrets. At first I tried to set up an environment secret and use that for my password entry, but my preliminary deploy tests failed. My next course of action was going the repository secret route. That proved successful. To set up a repository secret, simply click the 'new repository secret' button, enter your information, and click the 'add secret' button to save. I opted to save my server name and username as secrets too for good measure.

Once my secrets were taken care of, I updated the .yml file with the new info.

After commiting my .yml file, I made a basic update to my website's homepage to ensure the deploy was being triggered whenever changes were pushed to the GitHub repo. The 'actions' tab on the GitHub repo is the best way to monitor this activity.

A moment later, I was also able to see the new updates reflected directly on my website. This is a huge upgrade in terms of ease and efficiency. Definitely worth the initial effort!
